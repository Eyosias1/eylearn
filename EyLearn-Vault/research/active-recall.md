# Active Recall Research

> The evidence base behind [[active-recall]].

## Core concept

Active recall involves deliberate and effortful retrieval of information from memory in order to strengthen neural pathways and optimize long-term learning.

The key distinction is between **free recall** (active recall) and **cued recall** (flashcards). Cued recall only requires recognizing a connection between a term and a definition — it creates an illusion of knowing. Free recall requires synthesizing, reorganizing, comparing, applying, and contextualizing information, which creates deeper and more durable memory traces.

## Key studies

### Karpicke & Blunt (2011) — Retrieval Practice vs. Elaborative Encoding

**Experiment 1** — 80 undergraduates split into four groups:
1. Study (1x) — single study period
2. Repeated study (4x) — four consecutive study periods
3. Concept mapping — studied then created a concept map while viewing the text
4. Retrieval practice — studied then freely recalled as much as possible

Total learning time was matched between the concept mapping and retrieval practice groups. After one week, students were tested with verbatim and inference questions.

**Results:** Retrieval practice outperformed all three groups by a significant margin — 50% improvement over concept mapping collapsed across question types.

**Metacognitive finding:** Students predicted repeated study would yield the highest retention, and retrieval practice the worst. This illustrates the illusion of knowing created by encoding-based methods.

**Experiment 2** — 120 participants, studied either with concept mapping or retrieval practice. Tested after one week with both short-answer and concept map formats. Retrieval practice outperformed concept mapping on both test types, for both enumeration and sequence texts.

**Citation:** Jeffrey D. Karpicke, Janell R. Blunt, Retrieval Practice Produces More Learning than Elaborative Studying with Concept Mapping. *Science* 331, 772–775 (2011). DOI: 10.1126/science.1199327

## Six application methods (from the framework)

1. **Pretests prior to class** — quiz yourself on upcoming material using past papers and the syllabus before attending lecture. Expected to get most wrong — benefits from [[hypercorrection-effect]].
2. **In-class notes (the right way)** — write questions instead of transcribing. Convert slide titles into "What is..." questions. Use lecture objectives as free questions.
3. **Immediate review after class** — answer all questions from class verbally without notes. Color-code: black = correct, blue = partial, purple = couldn't answer. Darker shade = less confident.
4. **Feynman technique** — teach the concept to a fifth grader. Reveals hidden knowledge gaps. See [[feynman-technique]].
5. **Blurting method** — close the source, write everything you remember, then compare. See [[blurting]].
6. **Practice tests** — work through problem sets with all notes put away. For MCQ, explain why wrong answers are wrong, not just why the right answer is right.

## How it applies to eyLearn

- Questions presented with no hints or cues by default
- User must retrieve before seeing the answer
- Confidence rating before each answer surfaces the illusion of knowing — see [[metacognition]]
- [[ai-grading]] provides feedback without doing the retrieval for the user
- [[pre-test]] mode exploits the [[hypercorrection-effect]] before a study session
